>名字是一种艺术，它的艺术是把它们的名字和它们的意义联系在一起。就像说到苹果，我们会想到名为苹果的水果，会想到他是什么味道，我们可以通过一个抽象的名词，去进行具象化的联想，最终认知到，什么是苹果。可想而知正确的有意义的命名，是十分重要的。
>就好像作家在写作的时候，会根据故事情节，人物性格，身世背景，去赋予这个人物应该有的名字，读者可以从这个人物的名字中，就可以去分辨这个人物是一个怎么样的人，那么开发者同样如此，作者通过文字去创作故事，开发者通过编程语言去创造一段程序，两者其实有异曲同工之妙。
>因此，正确的，有意义的命名，可想而知其重要性。就好像如果作者无法为每一个角色安排好他们的位置，赋予他们符合故事走向的背景，那么最终这个故事可能走向烂尾，同样的，开发者无法为每个变量给与有意义，有价值的命名，那么最终这个程序，也可能走向被丢弃的命运。所以我们应该建立起正确的，有意义的给变量进行命名的意识，来让自己成为更好的开发者，来让自己的代码去创造更好的程序。

<h3>原则1:名副其实</h3>
<span>变量、函数、类的名称，应该显示出他们本身拥有的职责，可以让开发者一眼明白，他是做什么事情的代码。如果名称需要注释来补充，那就不算是名副其实。</span>
```ts
//变量名的命名
// bad
var d; // 消逝的时间，以日计
// 注释代替了命名的工作，假如没有注释，那么大概没人会知道这是什么

// good
let elapsedTimeInDays; // 经过的时间，以天计(只是翻译)
let daysSinceCreation; // 自创建以来的天数
let daysSinceModification; // 自修改以来的天数
let fileAgeInDays; // 文件的年龄
// 以上的命名则做到了命名应该做的事情，顾名思义，做到了身在其位，谋其职
```
